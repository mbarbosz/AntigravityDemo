# 🚀 Google Antigravity — Live Demo Experience

Bem-vindo ao repositório oficial da demonstração do **Google Antigravity**. Este projeto não é apenas um repositório de código comum, mas uma interface interativa e imersiva projetada para guiar uma apresentação ao vivo de **Agentic AI Coding** (Codificação Baseada em Agentes).

O objetivo desta demo é mostrar na prática a transição de um ecossistema de software legado de 1997 para uma infraestrutura moderna, escalável e segura em nuvem, utilizando o ecossistema Antigravity (`agy`) para orquestrar todo o processo de ponta a ponta sem digitação manual de código.

---

## 📂 Organização das Branches

Para facilitar a comparação visual e estrutural do impacto do agente entre o "antes" e o "depois", o repositório está organizado da seguinte forma:

* **Branch `main` (O Ponto de Partida):** Contém estritamente o código-fonte do **site legado**. É o cenário inicial da obra: HTML baseado em tabelas (`<table>`), tags obsoletas (`<font>`, `<center>`), zero responsividade e elementos gráficos antigos. É aqui que o agente inicia o mapeamento de contexto.
* **Branch `master` (A Apresentação):** Contém os arquivos desta **interface de demonstração interativa** (o arquivo HTML principal, os logos, o motor gráfico e os gráficos de eficiência). É o painel que o apresentador utiliza no navegador para conduzir o público ao vivo, contendo os prompts exatos, o roteiro passo a passo e as explicações técnicas.

---

## 🎯 Pilares Comprovados na Demo

A apresentação está estruturada para evidenciar três pilares que definem a próxima era do desenvolvimento de software:

1.  **Modernização Autônoma:** O agente consome diretrizes corporativas de design (através do arquivo `GEMINI.md`) e refatora o front-end legado autonomamente, gerando CSS moderno (HSL, Grid, Flexbox) e 100% responsivo.
2.  **Swarms Multi-Agentes (Orquestração):** Demonstra como o agente principal do terminal reconhece limites de competência e delega tarefas de design gráfico para um **subagente especialista em imagem**, que gera novos assets médicos via IA e os valida de forma assíncrona antes da aplicação final.
3.  **DevOps e Governança Serverless:** O agente gera toda a infraestrutura de containers (Dockerfile, Docker Compose, Nginx) e realiza o deploy no **Google Cloud Run**, mas só avança após passar por uma auditoria rigorosa controlada por **Agent Skills** (personas de Code Review sênior) e aprovação humana.

---

## 📈 Diferenciais Técnicos em Destaque

* **Context Caching Nativo:** Redução radical e achatamento na curva de custo por token, evitando o reprocessamento de codebases gigantes a cada iteração de debugging.
* **Fundamentação via MCP (Model Context Protocol):** Conexão do agente com fontes de dados e schemas reais para eliminar alucinações de arquitetura.
* **Fricção Controlada (Human-in-the-Loop):** Políticas declarativas de segurança que garantem que o humano revise o plano de ação e o diff antes de qualquer alteração física no código ou na nuvem.

## 🚀 Como Executar e Apresentar

1.  Certifique-se de que está posicionado na branch `master`.
2.  Clone este repositório localmente.
3.  Abra o arquivo de apresentação interativa (`demo.html`) diretamente em qualquer navegador moderno.
4.  Durante a apresentação, utilize as setas do teclado (`←` e `→`) para transicionar suavemente entre as 8 etapas do roteiro interativo.
